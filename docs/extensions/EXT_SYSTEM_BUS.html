System Bus Registry (pi.events)

Domain: Inter-Extension Communication

Overview

To avoid polling and decouple extensions from one another, Princess-Pi extensions communicate via the shared pi.events bus. This document acts as the authoritative registry for all established event channels and their payload contracts.

Event Channel Registry

clock:tick:1s

Emitted by: System Clock Service

Purpose: Fired every 1 second. Useful for precise countdowns or high-frequency polling monitors.

Payload: { tickCount: number, timestamp: number }

clock:tick:4s

Emitted by: System Clock Service

Purpose: Fired every 4 seconds. The standard UI refresh rate for TUI widgets (e.g., Serve widget) to prevent terminal flickering and excessive CPU load.

Payload: { tickCount: number, timestamp: number }

clock:tick:60s

Emitted by: System Clock Service

Purpose: Fired every 60 seconds. Intended for background maintenance, low-priority syncs, or garbage collection triggers.

Payload: { tickCount: number, timestamp: number }

Best Practices

  • Never leak listeners: Always capture the unsubscribe function returned by pi.events.on() and invoke it during the session_shutdown event.
  • Self-prune stale instances: Generate an instance ID on startup. If an event fires and your closure's instance ID doesn't match the module's active instance ID, immediately unsubscribe and return.
