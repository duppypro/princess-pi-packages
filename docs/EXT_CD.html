Dynamic Directory Switcher (cd-command.ts)

Domain: Session Workspace & Directory Orchestration

Overview

The Dynamic Directory Switcher provides dual-channel capabilities to change the active workspace directory of a running Pi session mid-stream without terminating the process or invalidating the model's prompt cache. It synchronizes both the human user and the AI agent inside the same workspace context by shifting the physical Node.js process working directory (process.cwd()), which in turn forces the TUI status bar to update in real-time.

Command & Tool Reference

👤 For Human (TUI Slash Command)

/cd <path>

Changes the user's current working directory dynamically. Path can be absolute, relative, or start with ~.

🤖 For Agent (AI custom tool)

change_working_directory

Instructs Pi to change its active execution directory. The LLM invokes this tool when instructed to switch focus to a different subdirectory/project.

Parameters Schema:

{
  path: Type.String({ description: "The target directory path (supports ~, relative, or absolute)" })
}

Behavioral Logic (Option A: Fail Fast)

Both interfaces share the same execution and safety engine under the hood, adhering to a strict Fail Fast policy:

  1. Path Resolution: The input path is parsed. If it starts with ~, it is resolved against the user's home directory (os.homedir()). If it is relative, it is resolved against the current process.cwd().
  2. Directory Validation:
    • The resolved path is checked for existence.
    • The resolved path is verified to be a directory (not a file).
    • If the path does not exist or is not a directory, it fails immediately. It alerts the human or AI with an error notification and does not change the directory.
  3. Directory Switch: Upon successful validation, the extension executes process.chdir(resolvedPath).
  4. Notification: Displays a success notification via the TUI UI showing the absolute target directory now active: 📁 Workspace changed to: <path>.

Synchronization Safeguards

To avoid context drift where the agent works in one directory but the physical process is in another, the global AGENTS.md mandates that:

  • The AI agent must never manually prefix paths with a different directory if instructed to switch focus. It must always call change_working_directory first.
  • This guarantees the physical CWD and the status bar are always 100% aligned with the active working context.
