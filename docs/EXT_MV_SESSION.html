Session Migrator (mv-session)

Domain: Session Workspace & Directory Orchestration

Overview

The Session Migrator command (/mv-session) provides a robust utility to permanently relocate an active Pi session's history and configuration to a new project directory mid-session. It moves the session's physical .jsonl file into the correct target project's session storage folder, re-writes the session's header block to update the metadata binding, and shifts the session manager and active process working directory to the target workspace.

Command & Tool Reference

👤 For Human (TUI Slash Command)

/mv-session <path>

Permanently moves the active session and CWD to the target directory. Path can be absolute, relative, or start with ~.

Behavioral Logic (Option A: Fail Fast)

The `/mv-session` command adheres to a strict Fail Fast policy to prevent file corruption or data loss:

  1. Path Resolution: Resolves the target directory path. Handles relative, absolute, and ~ paths.
  2. Target Directory Validation:
    • The resolved target path is checked for existence.
    • The resolved target path is verified to be a directory (not a file).
    • If the path does not exist or is not a directory, it fails immediately with an error notification. No file is moved.
  3. Active Session Verification:
    • Queries `ctx.sessionManager.getSessionFile()`.
    • If the session is ephemeral (not persisted on disk, e.g., started with `--no-session`), it simply shifts the workspace CWD (equivalent to /cd) and notifies the user that no file was moved.
  4. Destination Computation:
    • Generates the standard project-specific session directory for the target path:
      ~/.pi/agent/sessions/--home-user-path-to-target--/
    • Creates this target directory on disk if it does not exist yet.
  5. File Relocation:
    • Locates the active `.jsonl` file.
    • Reads the first line, parses the JSON session header, and rewrites the `cwd` field inside the header to match the target absolute path.
    • Moves the updated file to the new destination folder using standard Node.js file operations.
  6. Runtime Re-Binding:
    • Calls ctx.sessionManager.setSessionFile(newFilePath) to re-bind the active session to the new file path on disk.
    • Updates ctx.sessionManager.cwd = resolvedTargetPath to synchronize internal states.
    • Executes process.chdir(resolvedTargetPath) to shift the process directory.
  7. Success Notification: Displays a friendly confirmation showing the relocated path and updated CWD:
    📁 Session permanently moved and workspace changed to: <path>
