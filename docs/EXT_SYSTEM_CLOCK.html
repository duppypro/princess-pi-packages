System Clock Service (system-clock.ts)

Domain: Agent Infrastructure & Timing

Overview

A centralized, stateless timer service that emits standardized time-based domain events across the Pi pi.events bus. This eliminates the need for individual extensions to manage their own setInterval loops and prevents event-listener leaks.

Responsibilities

  • Maintain exactly one master interval during a session.
  • Cleanly tear down the interval on session_shutdown.
  • Emit multiple resolutions of ticks to allow extensions to subscribe to their desired frequency.

Event Bus Contract

  • clock:tick:1s - Fired every second.
  • clock:tick:4s - Fired every 4 seconds (Used by TUI widgets like Serve).
  • clock:tick:60s - Fired every 60 seconds (Used for background maintenance).

Payload: { tickCount: number, timestamp: number }
