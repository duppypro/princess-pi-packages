Spec: yada — Dynamic Streaming Log Deduplicator

A modern CLI utility designed to compress duplicate lines dynamically on standard output from real-time streaming inputs or static redirections.

2. Features & UX

In-place Deduplication

Uses ANSI escape sequences and carriage returns (\r) to update a status badge dynamically below the last unique log line.

Fuzzy/Near-Duplicate Matching

Decomposes lines into templates and variable slots (e.g., port numbers, timestamps).

Smart Range Accumulation

Collects slot values and formats them as continuous ranges (e.g., [3001-3003]).

Periodicity Detection

Tracks arrival intervals and uses autocorrelation to detect dominant frequencies, reporting messages like (every ~10s).

3. CLI Configuration

Usage: yada [options]

Options:
  -f, --format <string>  Custom collapse badge format (default: "☝️ +{count}")
  -s, --similarity <num> Similarity threshold 0.0-1.0 (default: 0.85)
  -w, --word-match       Use word-based template extraction
  -p, --no-periodicity   Disable advanced periodicity detection
  -g, --max-gap <num>    Max time gap in seconds (default: 10)
  --no-collapse          Disable in-place terminal rewriting
  -h, --help             Display help

4. Architectural Design

Built for O(N) linear performance, bypassing heavy Node readline in favor of raw process.stdin buffer processing.

5. Verification Plan

Automated benchmarks comparing chunk-buffering (O(N)) vs readline baseline (O(N2)) show ~32x speedup for 17k lines.
