Multi-Worktree Git Merger Utility (merge.ts)

Domain: Git Automation, Multi-Worktree Workflows & Step-Based Releases

Overview

The /merge extension provides a zero-token, local command to automate the validation and integration of completed feature branches from separate Git worktrees into the main branch. It ensures compliance with the Princess-Pi 5-Step Development Flow without invoking costly LLM reasoning turns.

CLI Mode (Outside Pi Coding Agent)

bin/merge.ts is a 1:1 standalone port of the same validation/merge/push sequence (plain execSync git calls, no Pi dependency). It's invokable as ./merge [commit-ish] from the repo root, or bare merge once the repo root is added to $PATH — including via Claude Code's ! prefix (e.g. !./merge 638e9b4), since Claude Code has no extension-dispatch mechanism that bypasses the model entirely the way Pi's registerCommand does. Behavior is identical to the Pi extension; only the output channel differs (console.log instead of ctx.ui.notify).

Verification Rules

Before any merge operations occur, the extension strictly enforces the following rules in sequence:

  1. No Self-Merge: Rejects running the command if the active worktree is already on the main branch.
  2. Clean Feature Worktree: Verifies that the current feature worktree contains no uncommitted (staged or unstaged) modifications.
  3. In Sync with Origin: Fetches origin and ensures the targeted commit-ish is present in the history of the remote tracking branch (preventing unpushed commits).
  4. Step 5 Validation: Validates that the targeted commit's message begins with Code and Spec Approved: (conforming to Step 5 of the Princess-Pi standard). If it does not, the command fails and automatically suggests the command to merge the most recent historical commit that does meet this standard.
  5. Clean Main Worktree: Finds the main worktree directory and verifies that it contains no uncommitted changes.

Merge and Push Execution

Once all validations pass, the utility performs the integration automatically inside the main worktree directory using git commands in that path:

  1. Switches/ensures the main worktree is on the main branch.
  2. Pulls the latest updates from origin/main.
  3. Merges the validated target commit into the main branch (leaving any subsequent unapproved commits on the feature branch).
  4. Pushes the newly merged main branch to origin.
  5. Logs absolute success and returns focus to the active worktree, ready for the next development task.
