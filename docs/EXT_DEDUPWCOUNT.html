dedupwcount Shell Command Spec

Goal: Compress noisy line logs for streams like tail -f ~/.pi-certs/logs/port-8080-access.log | dedupwcount.

Step 0: Make vs Buy

  • Buy candidates: uniq, awk, sed, and DevTools console grouping.
  • Gap: existing CLI tools do not provide live in-place +N duplicate updates plus numeric/letter monotonic range summaries.
  • Decision: make a focused local command that preserves shell pipe ergonomics.

Step 1: Better Name Ideas

  • logfold
  • linepulse
  • dupline
  • streamcompress
  • dedupwcount (kept as default for backward compatibility with issue request)

Step 2: Feature Spec (implemented subset)

  • Reads stdin line-by-line and emits compressed updates.
  • If consecutive lines match exactly, emits ☝️ +N updates.
  • If only numeric or single-letter tokens differ, treats them as collapsible and reports:
    • Range blocks ([3-16, 19-45] style)
    • Trend (up, down, mixed, flat)
    • Smoothness (smooth for step-1, otherwise gappy)
  • TTY mode rewrites the update line in place; non-TTY prints one update per line.

Step 3: Test Spec (implemented)

  • Exact duplicate collapse increments +1, +2, etc.
  • Numeric monotonic stream tracks ranges and gap behavior.
  • Single-letter monotonic stream tracks alphabetical ranges.
  • Pattern changes reset compression groups.
