Spec: wtft — Token Cost Auditing Widget

Provides real-time visibility into token consumption and associated monetary cost ($ USD) classified by work categories, running 100% locally with zero LLM costs.

1. Categorization Rules & Priority Hierarchy

Interactions are classified chronologically based on the file paths written or read, or the commands executed, under a strict priority system:

A. File Writes & Access (Highest Priority)

  • If two or more distinct categories are written, the turn is mixed.
  • If exactly one category is written, the turn is classified as that category (spec, code, tests, or research).

B. File Reads (Medium Priority)

  • If no writes exist but files are accessed, the same logic applies: multiple categories read becomes mixed, while a single category read becomes that category.

C. Command Analysis & Fallbacks (Lowest Priority)

If no files are accessed:

  • If a command is executing via the `bash` tool:
    • If starting with git -> classified as git.
    • If starting with grep, rg, ripgrep, or find -> classified as grep.
    • Any other commands (e.g. ls, node) -> other.
  • If no commands exist, but dialogue text is present -> prompt.
  • Otherwise -> other.

3. Command Reference

  • /wtft: Manages the horizontal stacked binned cost widget below the editor.
  • /wtft-other: Displays a cleanly aligned, descending-frequency histogram of "other" bash commands along with their cumulative token costs, scaled to fit your configured terminal width.
