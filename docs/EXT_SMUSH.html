EXT_SMUSH: The Interactive Rule Assimilator

A native TUI Extension for Pi Coding Agent. Resolves repo-specific instruction sets against user-global settings.

1. The Problem

When entering a third-party open source repository, developers are often confronted with a pre-existing CLAUDE.md or CONTRIBUTING.md. Automatically inheriting these files clutters the System Prompt with conflicting rules (like formatting styles or AI behavior). However, manually transcribing the good rules into a unified $CWD/AGENTS.md is tedious.

2. The /smush Solution

The /smush extension intercepts the user's workflow to offer an interactive "smushing" process. It reads local contextual files, parses them into discrete rules, and presents them via a TUI SettingsList. The user can interactively toggle which rules they wish to assimilate into their local $CWD/AGENTS.md.

3. Execution Flow

  1. User types /smush in the Pi input box.
  2. The extension searches the $CWD for CLAUDE.md, CONTRIBUTING.md, and AGENTS.md.
  3. It reads the files and uses a heuristic parser (extracting Markdown lists, headings, and bolded keywords) to separate the documents into actionable rules.
  4. The extension mounts a SettingsList TUI component replacing the editor.
  5. The user checks/unchecks the rules.
  6. Upon pressing Enter, the extension generates a clean $CWD/AGENTS.md containing only the selected rules.
  7. The extension prompts to optionally backup/delete the original CLAUDE.md to prevent Pi from double-loading it.

5. Heuristic Parsing

Because native Pi extensions run synchronously without guaranteed LLM-generate API access for simple scripts, /smush will employ a text-parsing heuristic:

  • Split file contents by \n.
  • Look for bullet points: lines starting with - or * .
  • If a bullet point is long enough (> 10 chars), consider it an actionable rule.

6. The Save Mechanism

When the user hits Enter:

// Generated $CWD/AGENTS.md
# Project Instructions (Repo-Specific)

These instructions override or append to the global standards for this specific repository.

## Assimilated Rules
- Use 2 spaces for indentation.
- Use Vite and Vitest for all build operations.
- Export components as default in src/lib.
