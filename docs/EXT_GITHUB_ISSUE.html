GitHub Issue Autocomplete (github-issue-autocomplete.ts)

Domain: Code Intelligence & External Integrations

Overview

The GitHub Issue Autocomplete extension hooks into the active Pi session to provide instant, fuzzy-searchable autocompletion for GitHub issues across multiple sibling repositories whenever the user types # in the prompt.

Core Architecture

1. Repository Discovery & Resolution:

On session startup, the extension auto-discovers all Git repositories residing at the same folder level as the CWD (by scanning sibling directories and synchronously parsing their .git/config files). This provides instant, O(1) local resolution without spawning slow child processes.

2. Two-Stage Autocomplete Modes:

  • Repo Selection Mode (digits not typed): Typing # triggers a dropdown list of all discovered sibling repositories (e.g. btw#, princess-pi-packages#), with the active CWD repository highlighted first. Arrow navigation dynamically updates the input text.
  • Issue Selection Mode (digits typed): As soon as the user types a digit after #, the extension parses the targeted repository (falling back to CWD repo if omitted) and triggers a background query to fetch its open issues via gh issue list --repo <owner/repo>. These are cached in-memory.

3. Title Expansion & Output:

When an issue is selected from the fuzzy-filtered suggestions list, the completed text inserts both the cross-repo issue reference and the double-quoted issue title, ending up with: repo-name#number "Issue Title" (e.g. princess-pi-packages#25 "BUG: TPM Rate Limiter widget...").
