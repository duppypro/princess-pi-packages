GitHub Issue Autocomplete (github-issue-autocomplete.ts)

Domain: Code Intelligence & External Integrations

Overview

The GitHub Issue Autocomplete extension hooks into the active Pi session to provide instant, fuzzy-searchable autocompletion for GitHub issues across multiple sibling repositories whenever the user types # in the prompt.

Core Architecture

1. Repository Discovery & Resolution:

On session startup, the extension auto-discovers all Git repositories residing at the same folder level as the CWD (by scanning sibling directories and synchronously parsing their .git/config files). This provides instant, O(1) local resolution without spawning slow child processes.

2. Two-Stage Autocomplete Modes:

  • Mode A (Explicit Repo Prefix): If the user types a repository prefix before the hash (e.g. btw#2 or btw#bug), the extension locks into that target repository and fuzzy-searches its open issues by the trailing characters (digits or letters).
  • Mode B (Root Hash): If the user simply types a hash (e.g. #, #2, #b):
    • If the user types letters or leaves it blank (e.g. #b), it displays both sibling repositories matching the letters AND fuzzy-matches CWD issues.
    • If the user types strictly digits (e.g. #25), it assumes the user wants an issue reference and strictly filters the CWD issues, suppressing the sibling repository list.

3. Title Expansion & Output:

When an issue is selected from the fuzzy-filtered suggestions list, the completed text inserts both the cross-repo issue reference and the double-quoted issue title, appending a trailing space so the user can continue typing seamlessly: repo-name#number "Issue Title" (e.g. princess-pi-packages#25 "BUG: TPM Rate Limiter widget..." ).

If a sibling repository is selected (e.g., btw#) instead of an issue, the completion strictly preserves the # character without appending a trailing space, placing the cursor directly after the hash so the user can immediately type digits for that repo's issues.
