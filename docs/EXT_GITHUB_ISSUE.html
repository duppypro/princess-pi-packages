GitHub Issue Autocomplete (github-issue-autocomplete.ts)

Domain: Code Intelligence & External Integrations

Overview

The GitHub Issue Autocomplete extension hooks into the active Pi session to provide instant, fuzzy-searchable autocompletion for GitHub issues whenever the user types # in the prompt. It dynamically parses the remote origin of the active Git worktree to determine the target repository.

Core Architecture

1. Lifecycle Hooking:

Instead of fetching issues on every keystroke, the extension hooks into the session_start event. It eagerly fetches up to 100 open issues in the background.

2. Dependency Validation (gh CLI):

The extension executes gh issue list --repo <repo> --json number,title,state. If the CLI is missing or unauthenticated, it presents a helpful error notification with installation instructions.

3. TUI Provider Layer:

Registers a custom AutocompleteProvider with ctx.ui.addAutocompleteProvider that matches tokens prefixed with # and fuzzy-filters the cached issue list in real-time.

Directory Sync Notes

Because the issue cache is primed on session_start using the active directory's remote repository, executing a /cd or /mv-session command mid-session to shift into a repository with a different remote origin will not automatically bust the issue cache. The user must issue a /reload command to trigger a new session_start event and re-hydrate the autocomplete list with the new repository's issues.
