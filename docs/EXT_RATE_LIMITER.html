Token Sentinel / Rate Limiter Utility (rate-limiter.ts)

Domain: Quota Management, Proactive Rate Limiting & Cooldown Breaks

Overview

The Token Sentinel / Rate Limiter extension prevents the agent from breaching model subscription quota limits (TPM - Tokens Per Minute). It runs transparently in the background, intercepting provider requests to check sliding-window usage metrics across all active local agent sessions. When a safety threshold is crossed, it enforces a 40-second synchronous "coffee break" (cooldown) to let API quota windows reset safely, notifying the user and maintaining a lockfile for external status bar / tmux integration.

Specification & Mechanisms

1. Quota Safety Limits

We maintain a hardcoded quota safety registry mapped to model shortcodes, configured to roughly 80% of actual provider subscription tiers to ensure a conservative buffer:

  • g3.5fla, glatfla, g3.5fli, glatfli: 2,500,000 TPM (Tier 2 limit: 3.0M)
  • g1.5pro, glatpro: 1,600,000 TPM (Tier 2 limit: 2.0M)
  • c3.5son, c3.5hai: 320,000 TPM (Tier 4 limit: 400K)
  • c3.0opu: 80,000 TPM

2. Active Session Log aggregation

The Sentinel scans the append-only .jsonl logs in ~/.claude/projects/ and ~/.pi/agent/sessions/ updated within the last 2 minutes to aggregate total TPM across all parallel agent sessions.

3. Schema & Token Counting (Issue #25 Fix)

To support multiple provider engines seamlessly, the token extractor parses provider-specific usage schemas from JSONL session records with robust fallbacks.

Due to Google's API integration logging structure, token usage is stored differently than Anthropic models. We aggregate both raw input tokens and cached prompt tokens (under Option B - Error on the side of more coffee breaks) to prevent exceeding active quota buckets:

Google/Gemini Logging Format: 
{
  "type": "message",
  "model": "gemini-flash-latest",
  "usage": {
    "input": 8572,
    "output": 405,
    "cacheRead": 25347,
    "totalTokens": 34324
  }
}
Anthropic/Claude Logging Format: 
{
  "type": "message",
  "message": {
    "role": "assistant",
    "model": "claude-3-5-sonnet-20241022",
    "usage": {
      "input_tokens": 5472,
      "output_tokens": 375
    }
  }
}

4. Parser Consolidation Rules

The parser retrieves sliding-window input tokens dynamically across either structure, summing cached reads when present:

let inputTokens = 0;
if (entry.usage) {
  if (typeof entry.usage.input === "number") {
    inputTokens = entry.usage.input;
    if (typeof entry.usage.cacheRead === "number") {
      inputTokens += entry.usage.cacheRead; // Sum Gemini Cached Reads
    }
  } else if (typeof entry.usage.input_tokens === "number") {
    inputTokens = entry.usage.input_tokens;
    if (typeof entry.usage.cache_read === "number") {
      inputTokens += entry.usage.cache_read;
    }
  } else if (typeof entry.usage.prompt_tokens === "number") {
    inputTokens = entry.usage.prompt_tokens;
  }
}

if (!inputTokens && msg?.usage) {
  const baseInput = msg.usage.input || msg.usage.input_tokens || msg.usage.input_token_count || msg.usage.prompt_tokens || 0;
  const cacheRead = msg.usage.cacheRead || msg.usage.cache_read || 0;
  inputTokens = baseInput + cacheRead;
}
